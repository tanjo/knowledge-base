# `git pull --rebase` → `git push` を1回で実行するエイリアス

`puresh` は、**今いるブランチ名を自動取得**して、次の順で実行します。

1. `git pull --rebase origin <現在ブランチ>`
2. 1 が成功した場合のみ `git push origin <現在ブランチ>`

`&&` でつないでいるため、`pull --rebase` が失敗（競合など）したときは `push` されません。

## 設定コマンド（そのまま実行）

```bash
git config --global alias.puresh '!f() { git pull --rebase origin "$(git rev-parse --abbrev-ref HEAD)" && git push origin "$(git rev-parse --abbrev-ref HEAD)"; }; f'
```

## 使い方

```bash
git puresh
```

## `.gitconfig` に直接書く場合

```ini
[alias]
  puresh = "!f() { git pull --rebase origin $(git rev-parse --abbrev-ref HEAD) && git push origin $(git rev-parse --abbrev-ref HEAD); }; f"
```

## 補足（運用でハマりやすい点）

- `detached HEAD` 状態だとブランチ名が取れず意図通り動きません。通常ブランチで実行してください。
- `rebase` 中に競合したら、解消後に `git rebase --continue` を実行し、最後に `git push` します。
- 「常に upstream へ push したい」運用なら `git push` のみ（`origin` 明示なし）にする別名の方が柔軟です。
